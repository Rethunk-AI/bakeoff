## Table → disk mapping

| Table | Stored on disk? | Own file or embedded? | Location (relative to `BAKEOFF_DATA_DIR`) | Notes |
|-------|----------------|----------------------|------------------------------------------|-------|
| `source_types` | Yes (seed) | Own file | `seeds/source_types.json` | Read-only seed asset; DB seeded from this file |
| `creators` | Yes (embedded) | In models file | `models/<uuid>.json` → `creator` object | 1-to-1 at write time; embedded to keep model file self-contained |
| `models` | Yes | **Own file** | `models/<uuid>.json` | Primary persistent record; UUID = UUID5(namespace, model_hash) or provisional |
| `model_sources` | Yes (embedded) | In models file | `models/<uuid>.json` → `sources` array | 1-to-many with model; embedded |
| `task_categories` | Yes (seed) | Own file | `seeds/task_categories.json` | Read-only seed asset |
| `tasks` | Yes | **Own file** | `tasks/<natural_key_hash>.json` | `natural_key_hash` = SHA256 of canonical path; stable across renames |
| `prompts` | Yes | **Own file** | `prompts/<content_sha256>.json` | `content_sha256` deduplication anchor; alias prompts use alias hash |
| `quantization_methods` | Yes (seed) | Own file | `seeds/quantization_methods.json` | JSON asset per bakeoff#13 decision; DB seeded from this |
| `runners` | Yes | **Own file** | `runners/<runner_id>.json` | Public key + metadata; one per registered runner |
| `runs` | Yes | **Own file** | `runs/<run_id>.json` | `run_id` = client-generated UUID |
| `run_model_metrics` | Yes (embedded) | In runs file | `runs/<run_id>.json` → `metrics` array | All metric rows for a run embedded; self-contained for distribution (bakeoff#13 §6) |

`run_queue` is excluded — agreed as ephemeral exception (job submissions, not persistent records).

---

## Embedding keys used

Where a table is embedded in another file's disk representation:

- `creator` in `models/<uuid>.json` — singular object (1-to-1)
- `sources` in `models/<uuid>.json` — array of `model_sources` rows (1-to-many)
- `metrics` in `runs/<run_id>.json` — array of `run_model_metrics` rows (1-to-many)

Data-carrying field names and types (text, numeric values, timestamps) are identical. No data field name diverges between disk and DB.
