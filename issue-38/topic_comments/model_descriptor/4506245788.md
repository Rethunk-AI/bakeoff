**Bastion response — 210825ZMAY26**

@gissf1 — correction acknowledged. Revised design:

**`model_sources` schema (final):**

| Column | Type | Notes |
|---|---|---|
| *(existing fields)* | | |
| `source_metadata` | JSONB NULLABLE | Provider identity + stats, flat structure — no sub-objects |
| `updated` | TIMESTAMPTZ NULLABLE | When all fields in this row (including `model_hash`) were sourced |

**`source_metadata` flat structure examples:**

HuggingFace:
```json
{"hf_commit": "abc123", "downloads": 50000, "likes": 1200}
```

Ollama:
```json
{"ollama_tag": "llama3:8b-q4_K_M", "ollama_digest": "sha256:...", "pulls": 3000}
```

No `stats` sub-object. No timestamp key inside the blob. Flat keys throughout. `updated` column is the single timestamp anchor for all fields in the row, including `model_hash`.

**Multi-query consistency protocol (recorded):**

1. All queries for a single row complete within a 1-minute window.
2. `updated` = earliest returned timestamp across all queries in the round.
3. If any query exceeds the 1-minute window: retry all queries.
   - Second round returns same values as first → treat as within window; use second round's `updated`.
   - Second round returns any different values → discard first round; restart from second round's data.
4. Hard limit: all queries must complete within 5 minutes total.

**Staleness queries** now use a native indexed column:
```sql
WHERE updated < NOW() - INTERVAL '30 days'
```

**bakeoff#14 design settled (final):**
- `ollama` added to `source_types`
- `source_metadata` JSONB — flat, provider identity + stats co-located, no sub-objects
- `updated` TIMESTAMPTZ column — single timestamp anchor for all sourced data in the row
- `model_hash` scoped to `updated` (traceable to that acquisition point)
- Multi-query consistency protocol as specified above

Ready for implementation sub-issues when directed.
