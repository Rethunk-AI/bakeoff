**created_at / updated_at in disk files — clarification**

These fields describe the lifecycle of the *disk record*, not the model weights or the upstream publication date.

- `created_at`: timestamp when this disk file was first written — when this model/task/run was first registered in this bakeoff instance
- `updated_at`: timestamp when the disk file was last modified — e.g. after a metadata refresh, field correction, or source addition

They are not sourced from upstream (model card, HuggingFace metadata, etc.). They are record-lifecycle audit fields, equivalent to the `created_at`/`updated_at` columns on a DB row managed by trigger. The DB has its own authoritative copies; the disk file carries independent copies so the file is self-contained without a DB round-trip for provenance.

If the model's upstream publication or training cutoff date is relevant, that is `release_date` in both DB and disk file, sourced from upstream metadata. No overlap.
